#### cmocka - современный фреймворк для [[модульное тестирование|модульного тестирования]] на языке с, написанный исключительно на с и не имеющий сторонних зависимостей

##### поддерживает макетирование функций через механизмы компоновщика и предоставляет богатый набор утверждений (assertions) для проверки результатов

##### элементы:
- **заглушка** - простая реализация функции или объекта, которая заменяет реальную зависимость модуля в процессе [[модульное тестирование|тестирования]]
- **макет** - объект-заместитель, который позволяет не только имитировать поведение зависимости, но и проверять, как именно тестируемый модуль взаимодействовал с ней. макеты настраиваются перед тестом: им указывается, какой вывод вернуть на конкретный ввод, а в конце теста проверяется количество и порядок вызовов
##### алгоритм макетирования функции в cmocka
- в коде теста определяется функция-обертка с префиксом `__wrap_`
- в обертке используется mock_type для получения значения из очереди макета
- в самом тесте используется will_return для помещения значения в очередь макета
- при сборке линковщику передается флаг --wrap=имя_функции
```C
// реализация макета для функции rand
int __wrap_rand() {
    return mock_type(int),
}

void test_example(void **state) {
    // указываем, что при следующем вызове rand вернет 10
    will_return(__wrap_rand, 10),
    assert_false(random_boolean()),
}
```
