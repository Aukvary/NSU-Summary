#### алгоритм
- массив хранится на диске, разбитый на блоки размера $b$
- алгоритм выполняет классический бинарный поиск, загружая блоки по мере необходимости
- когда область поиска сужается до размеров одного или нескольких блоков, поиск завершается во внутренней памяти без новых ios
- сложность составляет $o(\log_2(n/b))$, что существенно медленнее, чем поиск в [[B-дерево|b-дереве]]
```C
int binary_search_ext(const char *file, long left, long right, int x) {
    int block;
    while (left + 1024 <= right) {
        long mid = left + (right - left) / 2;
        read_block(file, mid, block);
        if (block < x) left = mid + 1;
        else if (block > x) right = mid - 1;
        else return binary_search_internal(block, x);
    }
    return -1;
}
```
