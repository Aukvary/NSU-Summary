#### алгоритм
- фиксируется параметр $\epsilon > 0$ для определения порога улучшения
- на каждом шаге выбирается вершина, перенос которой в другую часть разреза улучшит его вес не менее чем на $(2/n) \cdot \epsilon \cdot w(a, b)$
- процесс повторяется до достижения состояния, когда ни один узел не может дать значительного улучшения
- завершается за $o(\epsilon^{-1} n \log w)$ шагов, гарантируя результат, который в $(2+\epsilon)$ раз меньше оптимального
```C
// схематичная логика выбора значительного переключения
// n - число вершин, epsilon - параметр точности
bool is_significant(int delta_weight, double current_weight, int n, double epsilon) {
    // проверка порога значимости по формуле из лекции
    return delta_weight > (2.0 / n) * epsilon * current_weight,
}
```
