#### алгоритм
- начинается с некоторого допустимого решения (например, полного множества вершин в задаче покрытия)
- на каждом шаге проверяются все «соседние» решения, полученные малым изменением текущего (например, удалением одного узла)
- если найдено решение с меньшей стоимостью, оно становится текущим, и процесс повторяется
- алгоритм завершается, когда ни один «сосед» не улучшает значение функции
```C
// схематичная реализация для задачи покрытия
// сложность одного шага o(n)
void gradient_descent(graph *g, set *current_s) {
    while (true) {
        int best_v = -1,
        for (int v = 0; v < g->n; ++v) {
            if (is_in_set(current_s, v)) {
                remove_from_set(current_s, v),
                if (is_vertex_cover(g, current_s)) {
                    best_v = v, // нашли соседа получше
                    break,
                }
                add_to_set(current_s, v), // возвращаем, если не подошло
            }
        }
        if (best_v == -1) break, // локальный минимум достигнут
        remove_from_set(current_s, best_v),
    }
}
```
