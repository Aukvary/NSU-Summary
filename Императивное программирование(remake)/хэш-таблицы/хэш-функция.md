#### хэш-функция - это математическая формула, которая преобразует входной ключ в целое число, используемое в качестве индекса в [[хэш-таблица|хэш-таблице]] 

##### свойства хорошей хэш-функции
- она должна распределять элементы по массиву максимально равномерно, чтобы избежать [[коллизия|коллизии]] 
- вычисление должно быть быстрее, чем выполнение $\log n$ сравнений ключей
- для одного и того же входного значения функция всегда должна возвращать одинаковый результат 

##### примеры простых хэш-функций 
 - **метод деления:**
	 - хэш-значение это остаток от деления ключа $x$ на размер таблицы $M$ 
	 - формула $h(x) = x \mod M$ 
	 - для более эффективной работы обычно в качестве $M$ обычно выбирают простое число
- **метод умножения:**
	- ключ $k$ умножается на константу $A(0 < A \lt 1)$ 
	- извлекается дробная часть полученного произведения 
	- дробная часть умножается на размер таблицы $M$ 
	- оптимальным значением $A$ считается $\frac{\sqrt{5} - 1}{2} = 0,6180339887$ 
- **метод середины квадрата середины квадрата:**
	- значение ключа возводится в квадрат
	- из середины квадрата извлекается $r$ цифр $(1234^2 \rightarrow 1522756, r = 2 \rightarrow 27)$ 
- **метод сворачивания:**
	- ключ разбивается на несколько частей равной длины(кроме последней)
	- все части складываются
	- хэш-значение получается из этой суммы путём игнорирования последнего переноса

##### типы хэширования
- **универсальное хэширование**
	- метод выбора хэш-функции случайным образом из специально разработанного семейства функций в начале выполнения программы 
	- **проблема:** если использовать одну фиксированную хэш-функцию, то злоумышленник может подобрать такие ключи, которые все попадут в одну ячейку
	- рандомизация гарантирует, что никакие входные данные не будут приводить наихудшему случаю  
	- семейство функций $\chi$ называется универсальным, если для любой пары различных значений вероятность их коллизии при случайном выборе функции из $\chi$ не превышает $1/m$, где $m$ - размер таблицы
	- **пример:** $h_{a, b} = ((ax + b) (\text{mod } p))(\text{mod } m)$, где $p$ - простое число, большее любого возможного ключа, а $a, b$ случайны коэффициенты 
- **идеальное хэширование**
	- метод, обеспечивающий поиск в таблице за время $O(1)$ в наихудшем случае
	- это достижимо, если набор ключей является статическим, то есть фиксированный и известен заранее 
	- для реализации идеального хэширования используется двухуровневая схема хэширования
		- **первый уровень:** аналогичен хешированию с цепочками, $n$ ключей распределяются по $m = n$ ячейкам основной таблицы с помощью универсального хеширования 
		- **второй уровень:** вместо связных списков в каждой ячейке создаётся вторичная хеш-таблица
		- **исключение коллизий:** чтобы гарантировать отсутствие коллизий во вторичной таблице, её размер $m_j$ устанавливается равным квадрату числа ключей $n_j$ попавших в данную ячейку
		- несмотря на квадратичный размер вторичных таблиц, математическое ожидание остаётся линейным(в среднем менее $2n$)
