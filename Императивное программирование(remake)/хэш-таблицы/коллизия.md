#### коллизия - ситуация, при которой [[хэш-функция]] сопоставляет два или более различных ключа одной и той же ячейке памяти в [[хэш-таблица|таблице]] 

##### методы разрешения с открытой адресацией
- **линейное зондирование**
	- принцип: если ячейка $h(k)$ занята, проверяются следующие ячейки по порядку $(h(k) + 1) \mod M, (h(k) + 2) \mod M$ 
	- плюс - простота реализации
	- минус - длинные цепочки замедляют поиск(первичная кластеризация)
- **квадратичное зондирование**
	- принцип: если ячейка, определённая основной хэш-функцией $h^\prime(k)$, уже занята, то новая позиция ищется по формуле $h(k, i) = [h^\prime(k) + c_1i + c_2i] (mod\ M)$, где  $c_1, c_2$ какие-то коэффициенты, а $i$ "попытка" найти позицию
	- плюс - устойчив к первичной кластеризации
	- минусы - для эффективного заполнения таблицы требуется тщательный подбор коэффициентов и размера таблицы
- **двойное хэширование**
	- принцип: используется две независимые хэш-функции, шаг вычисляется с помощью второй хэш-функции по формуле $h(k, i) = [h_1(k) + ih_2(k)](mod\ M)$, где $i$ - номер шага
	- плюс - избегает кластеризации
	- минус - сильно зависит от того, чтобы размер таблицы был простым числом
##### метод разрешения с закрытой адресацией
- **метод цепочек**
	- принцип: каждая ячейка таблицы содержит не сам элемент, а указатель на заголовок [[связный список|связанного списка]] со всеми элементами
	- плюс - полностью избегает проблему с кластеризацией 
	- минус - затраты памяти на связный список
