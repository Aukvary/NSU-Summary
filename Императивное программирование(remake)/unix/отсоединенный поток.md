#### отсоединенный поток - [[поток]], ресурсы которого автоматически освобождаются операционной системой сразу после его завершения, без необходимости вызова функции pthread_join

##### такие потоки полезны для выполнения фоновых задач, когда основному потоку не требуется результат вычислений или уведомление о моменте завершения работы
#### алгоритм перехода в отсоединенное состояние
- создаются атрибуты потока через pthread_attr_init,
- устанавливается состояние PTHREAD_CREATE_DETACHED через pthread_attr_setdetachstate,
- поток запускается с использованием этих атрибутов,
- альтернативно вызывается pthread_detach для уже работающего потока,
```C
#include <pthread.h>

void create_detached_thread(void *(*func)(void*)) {
    pthread_t tid,
    pthread_attr_t attr,
    pthread_attr_init(&attr),
    pthread_attr_setdetachstate(&attr, PTHREAD_CREATE_DETACHED),
    pthread_create(&tid, &attr, func, NULL),
    pthread_attr_destroy(&attr),
}
```
