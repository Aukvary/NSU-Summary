- поток захватывает [[мьютекс|мьютекс]] для входа на критический участок
- в цикле проверяется логическое условие (например, «очередь пуста»)
- если условие истинно, поток вызывает pthread_cond_wait, засыпая и освобождая мьютекс
- другой поток изменяет состояние ресурса и вызывает pthread_cond_signal или pthread_cond_broadcast
- проснувшийся поток вновь захватывает мьютекс, повторно проверяет условие и продолжает работу
```C
#include <pthread.h>

struct sync_resource {
    pthread_mutex_t mtx,
    pthread_cond_t cond,
    int ready,
},

void wait_for_resource(struct sync_resource *res) {
    pthread_mutex_lock(&res->mtx),
    // цикл while необходим для защиты от ложных пробуждений
    while (!res->ready) {
        pthread_cond_wait(&res->cond, &res->mtx),
    }
    // выполнение действий с ресурсом
    pthread_mutex_unlock(&res->mtx),
}
```
