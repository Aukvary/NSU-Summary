- для всех вершин путь до стартовой равняется $\infty$, а предшественник равен NULL
- алгоритм выполняется $|V| - 1$ проходов 
- в каждом проходе рассматривается каждое ребро графа, и для него выполняется процедура релаксации
	- если найден путь к вершине $v$ через вершину $u$ короче, чем известные ранее $v.d > u.d + w(u, v)$, то значение обновляется 
- проверка на отрицательные циклы $v.d > u.d + w(u, v)$ 

##### сложность
- суммарная сложность $O(|V| \cdot |E|)$ 

##### корректность 
- если граф $G$ не содержит отрицательных циклов, достижимых из $s$, то по завершении $|V| - 1$ итераций основного цикла для всех вершин $v \in V$ выполняется точное равенство $v.d = \delta(s, v)$
- если в графе существует достижимый из $s$ цикл отрицательного веса, алгоритм возвращается false
- **случай отсутствия отрицательных циклов:**
	- пусть вершина $v$ достижима из источника $s$ 
	- тк в графе отсутствуют достижимые циклы отрицательного веса, среди всех путей из $s$ в $v$ существует хотя бы один истинный кратчайший путь $p = <s, v_2, ..., v>$, данный путь гарантированно является простым , а значит количество рёбер $k$ не превосходит $|V| - 1$ 
	- согласно фундаментальному свойству релаксации пути, если рёбра кратчайшего пути релаксируются в порядке их следования на пути (пусть даже с промежуточными релаксациями других рёбер), то после $i$-й направлений релаксации оценка расстояния для узла $v_i$ становится точной $v_i.d = \delta(s, v_i)$ 
	- поскольку на каждой $|V| - 1$ итераций алгоритма релаксации подвергаются абсолютно все рёбра графа, то 
		- на 1-й итерации гарантированно будет отрелаксировано ребро $(s, v_1)$ следовательно $v_1.d = \delta(s, v_1)$ 
		- на 2-й итерации будет отрелаксировано ребро $(v_1, v_2)$, откуда $v_2.d = \delta(s, v_2)$ 
			- на k-й итерации будет отрелаксировано финальное ребро на пути $(v_{k-1}, v_k)$, и значит значение целевой вершины станет точным $v.d = \delta(s, v)$
	- тк $k \le |V| - 1$, то за полное число итерация все вершины гарантировано получат свои точные кратчайшие расстояния 
	- по свойству неравенства треугольника для кратчайших путей всегда справедливо соотношение $\delta(s, v) \le \delta(s, u) + w(u, v)$
	- поскольку к концу работы цикла все оценки точны, ни для одного ребра графа на этапе проверки не сможет выполниться неравенство $v.d > u.d + w(u, v)$
- **случай наличия:**
	- пусть в графе из источника $s$ достижим некоторый ориентированный цикл $c = <v_0, v_1, ..., v_k>, v_0 = v_k$, суммарный вес рёбер которого строго меньше нуля 
	- допустим алгоритм не заметил этого цикла и вернул true, это значит, что на этапе финальной проверки ни для одного ребра этого цикла не сработало условия выхода, то есть для всех рёбер цикла выполнилось неравенство $v_i.d \le v_{i-1}.d + w(v_{i-1}, v_i)$ 
	- просуммируем эти $k$ неравенств по всему контору $$\sum_{i=1}^k v_i.d \le \sum_{i=1}^kv_{i-1}.d + \sum_{i=1}^kw(v_{i-1}, v_i)$$
	- тк цикл замкнут, множество вершин в левой и правой частях сумм абсолютно идентичны, каждая вершина вносит одинаковый вклад как в левую так и в правую части, а следовательно мы можем вычесть $\sum_{i+1}^kv_i.d$ из обеих частей неравенства, в результате получаем $$0 \le \sum^k_{i=1}w(v_{i-1}, v_i)$$
	- это выражение противоречит условию того, что вес цикла является отрицательным
