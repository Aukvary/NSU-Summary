#### lsm-дерево - многоуровневая структура данных, оптимизированная для интенсивной записи во [[внешняя память|внешнюю память]]

##### свойства
- **компонент $C_0$:** находится в ОЗУ, обычно это небольшое сбалансированное дерево, куда попадают все новые данные
- **компоненты $C_1, ..., C_k$:** находятся на диске и являются иммутабельными, каждые следующий уровень обычно в $f$ ($f \geq 2$)раз больше предыдущего
- когда буфер $C_0$ заполняется, его содержимое сортируется и сбрасывается на диск в виде нового сегмента 
- может ускорить алгоритм используя [[фильтр блюма]], который позволяет несколько раз не использовать операцию IO

##### слияния
- **поуровневое**
	- на каждом уровне поддерживается только один большой отсортированный
	- при слиянии данные с уровня $i$ полностью переписываются вместе с данными уровня $i+1$
	- **плюс:** быстрый поиск (мало компонентов для проверки)
	- **минус:** высокое усиление записи, тк один и тот же элемент переписывается $O(f)$ раз для спуска на один уровень
- **поярусное:**
	- на каждом ярусе может быть до $f$ независимых сегментов
	- слияние происходит только тогда, когда ярус полностью заполнен: все $f$ сегментов сливаются в один сегмент на следующем ярусе
	- **плюс:** очень быстрая запись $1/B$ операций
	- **минус:** медленный поиск, тк нужно проверять много сегментов на каждом ярусе
