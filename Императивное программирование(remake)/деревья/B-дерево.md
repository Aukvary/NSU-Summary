#### b-дерево - оптимальная для внешней памяти [[структура данных]], представляющая собой [[(a, b)-дерево]], где размер узла совпадает с размером блока $b$, а минимальное ветвление $a = \lceil b/2 \rceil$

##### позволяет выполнять поиск, вставку и удаление за $o(\log_b n)$ операций ввода-вывода, что на порядки эффективнее обычного двоичного поиска из-за огромного коэффициента ветвления

##### свойства
- каждый узел имеет максимум $m$ потомков и $m - 1$ ключей
- **минимальное заполнение:** каждый узел должен иметь не менее $m / 2$ потомков
- **свойство корня:** корень имеет не менее двух потомков, если он не являться листом 
- все листья находятся на одной уровне 
- ключи внутри внутри каждого узла хранятся в возрастающем порядке, разделяя указатели на поддеревья так, что значения в поддереве межу $K_i$ и $K_{i+1}$ находятся в диапазоне $[K_i, K_{i + 1}]$ 

##### преимущества
- Размер узла B-дерева обычно совпадает с размером дискового блока, за одно обращение к диску (один IO) считывается целый узел, содержащий сотни или тысячи ключей
- благодаря огромной степени ветвления (b от 50 до 2000) дерево получается очень «плоским». Даже для миллиардов записей высота дерева составляет всего 3–4 уровня
- поиск конкретного ключа требует количества чтений с диска, равного высоте дерева (O(logb​n)), что в сотни раз быстрее, чем бинарный поиск по неиндексированным данным во внешней памяти
#### сигнатура
```C
// структура узла b-дерева
struct b_node {
    int count, // количество ключей в узле
    int keys, // ключи (размер блока b)
    long children, // смещения дочерних узлов в файле
    bool is_leaf,
},
```
#### поиск
```C
// поиск за o(log_b n) ios
long b_tree_search(long node_addr, int key) {
    struct b_node node,
    read_from_disk(node_addr, &node), // один ввод-вывод
    int i = 0,
    while (i < node.count && key > node.keys[i]) i++,
    if (i < node.count && key == node.keys[i]) return node_addr,
    if (node.is_leaf) return -1,
    return b_tree_search(node.children[i], key),
}
```
