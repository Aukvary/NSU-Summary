## Установка:
- ### git:
	- перейдите по [ссылке](https://git-scm.com/install/windows) и установите git для вашей операционной системы
	- откройте терминал и пропишите
		- Windows: `git config credential.helper`, если не выдало `manager`, то пропишите `git config set credential.helper manager` 
		- MacOS: `git config --global credential.helper osxkeychain`
	- откройте терминал и пропишите
		``` Bash
			cd path/to/directory #перейти к директории, в которую вы хотите склонировать репозиторий
		  
			git clone https://github.com/Aukvary/NSU-Summary.git #склонировать репозиторий
		```
- ### obsidian:
	- перейдите по [ссылке](https://obsidian.md/) и скачайте obsidian
	- после установки выберете открыть `открыть папку как хранилище` и выберете директорию, которую вы склонировали
	- первоначальная настройка: 
		- файлы и ссылки -> место для вложенных файлов по умолчанию -> в той же папке, где расположен файл
		- файлы и ссылки -> всегда обновлять внутренние ссылки -> включить
		- встроенные плагины:
			- Шаблоны - позволяет создавать файлы-шаблоны. установите папку `Templates` как источник
	- установка плагинов:
		- откройте настройки -> сторонние плагины -> включить сторонние плагины
		- важные плагины:
			- Git - для удобного управления 
			- DataViewer - позволяет делать списки/таблицы из файлов через SQL-подобные запросы
		- рекомендации:
			- поставьте приятную тему, которая будет раскрашивать заголовки в разные цвета, чтобы было легко ориентироваться по файлу `я использую topaz + style settings(плагин) + topaz nord(тема внутри topaz)`
			- Advanced Tables - позволяет удобно работать с таблицами
### Архитектура
- Templates - служебная папка 
- папка предмета(например дискретная математика) - папка, в которой лежит информация по этому предмету
	- внутри находится заголовочный файл, который структурирует всю информацию по теме и содержит общую информацию
		- внутри содержаться заголовки отвечающие за определённые разделы и структурированная информация с помощью плагина `DataViewer` 
	- папки, которые хранят информацию по определённой разделу
		- внутри лежат файлы, которые отвечают за конкретную информацию по разделу
		- вот примерная структура файлов, тэги должны отвечать за предмет и раздел, например `#матанализ` `#векторы`, *** нужны для отделения информации от тэгов
			```
				#### info header
				***
				#tag1 #tag2
			```
