- **база рекурсии:**
	- проверить, является ли текущий граф полным
	- если в графе нет ни одной пары смежных вершин, то граф полный
	- для полного графа [[хроматическое число]] равно количеству вершин
- **шаг рекурсии**
	- найти любую пару смежных вершин $v, w \in V$ 
	- сконструировать графы $G+e$ и $G/e$
	- рекурсивно вызвать функцию вычисления хроматического числа для обеих ветвей 
	- найти и вернуть минимум из графов 
- **доказательство конечности**
	- введём в качестве меры прогресса величину $n_a(G)$ - количество неупорядоченных пар несмежных вершин в графе, для графа с $n$ вершинами и $m$ рёбрами $$n_a(G) = \begin{pmatrix}n \\ 2\end{pmatrix} - m$$
	- **ветвь G+e**
		- количество вершин остаётся неизменяемым, а количество рёбер увеличивается $$n_a(G + e) = \begin{pmatrix}n \\ 2\end{pmatrix} - (m+1) = n_a(G + e) - 1$$
	- **ветвь G/e**
		- количество вершин уменьшается на 1, число рёбер становится равным $m - |N(v) \cap N(w)|$, где $|N(v) \cap N(w)|$ - общие соседи
		- простой комбинаторный переход показывает, что число несмежных пар в графе $G/e$ всегда строго меньше $$n_aF(G/e) < n_a(G)$$
- тк $n_a(G)$ является целым неотрицательным числом и строго уменьшается при любой ветви, то значение сходится 

##### временная сложность
- поскольку на каждом шаге алгоритм порождает два рекурсивных вызова, дерево вычислений является бинарным
- высота этого дерева ограничена величиной $n_a(G) \le \begin{pmatrix}n\\2\end{pmatrix} = O(n^2)$ 
- в худшем случае $$O(2^{n_a(G)}) \text{ или } O(2^{\frac{n(n-1)}{2}})$$
